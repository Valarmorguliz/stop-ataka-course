# Инструкция по развёртыванию на Vercel

Этот проект представляет собой Next.js приложение, готовое к развёртыванию на Vercel.

## ✅ Проект уже настроен для Vercel!

Вы подключили репозиторий на сайте Vercel — теперь всё готово для автоматического деплоя.

## Что уже сделано:

1. **Настроен `next.config.ts`** — Vercel автоматически определит оптимальные настройки
2. **Создан `vercel.json`** — минимальная конфигурация для корректной работы маршрутизации
3. **Готовый `package.json`** — содержит все необходимые скрипты для сборки

## Как работает автоматический деплой:

После подключения репозитория в интерфейсе Vercel:

- ✅ **Каждый пуш в `main` ветку** → автоматически деплоится на production
- ✅ **Pull Request'ы** → создают preview-деплои с уникальным URL для тестирования
- ✅ **Другие ветки** → также получают preview-деплои

## Шаг 1: Завершите настройку в интерфейсе Vercel

1. В личном кабинете Vercel выберите ваш проект
2. Перейдите в **Settings** → **Build & Development Settings**
3. Убедитесь, что настройки следующие:
   - **Framework Preset**: Next.js (определится автоматически)
   - **Build Command**: `npm run build`
   - **Output Directory**: `.next` (по умолчанию для Next.js)
   - **Install Command**: `npm install`
4. Если вы ещё не деплоили проект, нажмите **Deploy**

## Шаг 2: Настройка переменных окружения (если требуется)

Если вашему проекту нужны переменные окружения:

1. В дашборде Vercel выберите ваш проект
2. Перейдите в **Settings** → **Environment Variables**
3. Добавьте необходимые переменные
4. Сохраните и сделайте редиплой проекта

## Шаг 3: Настройка домена (опционально)

1. В дашборде проекта перейдите в **Settings** → **Domains**
2. Добавьте ваш домен
3. Настройте DNS-записи у вашего регистратора домена:
   - **A record**: `76.76.21.21`
   - **CNAME**: `cname.vercel-dns.com` (для поддоменов)

## Особенности данного проекта

Проект использует стандартный режим Next.js с поддержкой всех возможностей:
- ✅ Серверное рендерирование (SSR)
- ✅ Статическая генерация (SSG)
- ✅ API Routes
- ✅ Динамические маршруты
- ✅ Оптимизация изображений

## Файл vercel.json

В проекте создан файл `vercel.json` с минимальной конфигурацией:
```json
{
  "framework": "nextjs",
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/"
    }
  ]
}
```

Эта конфигурация обеспечивает корректную работу клиентской маршрутизации Next.js.

## Проверка перед деплоем (опционально)

Если хотите протестировать локально:

```bash
# Установите Vercel CLI
npm install -g vercel

# Войдите в аккаунт
vercel login

# Запустите локальный превью
vercel dev
```

Или просто запустите проект в режиме разработки:
```bash
npm install
npm run dev
```

## Что делать, если что-то не работает

1. **Проверьте логи билда** в дашборде Vercel (**Deployments** → выберите деплой → **View Build Logs**)
2. **Убедитесь, что зависимости установлены** — Vercel автоматически выполняет `npm install`
3. **Проверьте переменные окружения** — если они используются в коде
4. **Посмотрите ошибки в консоли браузера** после деплоя
