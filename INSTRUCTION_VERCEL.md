## 📋 Пошаговая инструкция

### Шаг 1: Отправьте изменения в ваш репозиторий GitHub

Откройте терминал в папке проекта и выполните команды:

```bash
# Если вы ещё не добавили удалённый репозиторий
git remote add origin https://github.com/ВАШ_USERNAME/ВАШ_РЕПОЗИТОРИЙ.git

# Проверьте текущую ветку
git branch

# Если вы в ветке qwen-code-..., переключитесь на main или master
git checkout main
# или
git checkout master

# Влейте изменения из текущей ветки (если нужно)
git merge qwen-code-29f94e12-97c9-4085-a201-ea97861ef846

# Отправьте изменения в GitHub
git push -u origin main
# или для master ветки:
# git push -u origin master
```

**Если у вас нет локального git**, просто скопируйте эти файлы в ваш репозиторий через интерфейс GitHub:
- `next.config.ts` (обновлённый)
- `vercel.json` (новый файл)

---

### Шаг 2: Настройте проект в Vercel

1. **Зайдите на [vercel.com](https://vercel.com)**
2. **Войдите через GitHub** (рекомендуется)
3. **Нажмите "Add New..." → "Project"**
4. **Найдите ваш репозиторий** в списке и нажмите "Import"
5. **Настройте проект**:
   - **Framework Preset**: Next.js (определится автоматически)
   - **Root Directory**: оставьте как есть (`./`)
   - **Build Command**: `npm run build` (по умолчанию)
   - **Output Directory**: `.next` (по умолчанию)
6. **Нажмите "Deploy"**

---

### Шаг 3: Проверьте результат

После деплоя:
- Вы получите URL вида `https://ваш-проект.vercel.app`
- Проект будет автоматически обновляться при каждом пуше в главную ветку
- Pull Request'ы будут создавать preview-деплои для тестирования

## ⚠️ Возможные проблемы и решения

### Проблема 1: Ошибка сборки "Node.js 20 actions are deprecated"
**Это предупреждение GitHub Actions, а не ошибка Vercel.**

Vercel использует собственную инфраструктуру и не зависит от GitHub Actions. Эта ошибка появляется только если у вас остались старые workflow-файлы для GitHub Pages.

**Решение**: Удалите папку `.github/workflows/` если она существует:
```bash
rm -rf .github
```

### Проблема 2: Ошибка "Module not found" при сборке
**Причина**: Отсутствуют зависимости

**Решение**: Убедитесь, что все зависимости указаны в `package.json`:
```bash
npm install
git add package.json package-lock.json
git commit -m "Fix dependencies"
git push
```

### Проблема 3: Не работают динамические роуты (например, `/course/[courseId]`)
**Причина**: Неправильная конфигурация

**Решение**: Файл `vercel.json` уже содержит правильные настройки rewrites. Просто убедитесь, что он загружен в репозиторий.

### Проблема 4: Ошибки с переменными окружения
**Причина**: Проект использует переменные окружения

**Решение**: 
1. Зайдите в проект на Vercel
2. Перейдите в **Settings** → **Environment Variables**
3. Добавьте необходимые переменные:
   - `NEXT_PUBLIC_*` - для клиентских переменных
   - Другие - для серверных

## 🎯 Как работает автоматический деплой

После настройки:

1. **Пуш в главную ветку** → Production деплой
   - Обновляется основной сайт
   - Получает уникальный URL

2. **Pull Request** → Preview деплой
   - Создаётся временная версия для тестирования
   - URL вида `https://ваш-проеGit-HASH.vercel.app`

3. **Пуш в другую ветку** → Development деплой
   - Для тестирования функций
